# xlsx2md 要件定義書 v1.0

Excel で作成された文書（アプリケーション要件定義書・画面設計書）を Markdown に変換する Python モジュールの要件。

## 1. 背景・目的

- 対象 Excel は多くの場合、アプリケーションの**要件定義書**および**画面設計書**。
- Excel を方眼紙（レイアウトキャンバス）として使っており、**セル結合からなる複雑なテーブル**と**画像添付**が多用される。
- 1 つの xlsx に複数シートがあり、**各シートは独立した文書**。シートごとに独立した Markdown として出力する。
- 出力 Markdown の主用途は **LLM への入力**。オリジナルのレイアウトを可能な限り**忠実に再現**することを優先する。

## 2. 基本方針

- **汎用ヒューリスティック**：対象はテンプレート不定（様式バラバラ／不明）。意味推定（フォーム／データ表の判別等）は誤判定リスクが高いため最小化し、**グリッドをそのまま写し取る**方針を採る。
- **忠実再現優先**：LLM は rowspan/colspan 付き HTML テーブルを解釈できるため、Markdown 装飾よりも**グリッド位置の保存**を価値とする。
- **頑健性 > 精度**：未知の構造でも破綻しないことを最優先する。

## 3. 機能要件

### 3.1 出力単位
- xlsx 1 ファイル → **シートごとに独立した .md** を出力する。
- シートの並び順を保持する。

### 3.2 シートの表現
- 1 シート = 原則 **1 つの HTML `<table>`**。使用領域のグリッドをそのまま `<table>` 化する。
- 各 .md の先頭にシート名を見出し（`# {シート名}`）として出力する。

### 3.3 セル結合
- `merged_cells` を **colspan / rowspan** に変換する。
- 結合範囲の左上セルに値を出力し、被覆される他セルは出力しない。

### 3.4 トリミング
- 使用領域の**外周の空白行・空白列のみ除去**する。
- 内部の空セル・空行・空列は**レイアウトとして保持**する（空セルは空の `<td>`）。
- 隠し行／隠し列も**出力に含める**（情報欠落ゼロを優先）。フラグで除外可能とする。

### 3.5 セル値の取得
- **数式**：キャッシュ値（表示値）を採用（`data_only=True`）。
  - キャッシュが存在しない場合（一度も Excel で保存されていないファイル）は、**式文字列にフォールバック**する。
- **日付・数値**：`number_format` 適用後の表示文字列を採用する（例 `2026/06/15`）。raw シリアル値は使わない。
- **セル内改行**：`<br>` に変換する。
- **ハイパーリンク**：`テキスト (URL)` 形式で URL を併記する。

### 3.6 書式
- **文字値・セル結合・セル内改行のみ**再現する。
- 背景色・太字・罫線・フォント等の見た目書式は**破棄**する（LLM 入力としてノイズを避ける）。

### 3.7 画像
- 埋め込み画像（ピクチャ）をファイルに**抽出**する。
- 本文にはリンクは張らず、**プレースホルダ**を出力する。
  - 形式：`[画像: {ファイル名} | アンカー {セル番地}]`（例：`[画像: image_3.png | アンカー B12]`）。
- 画像はセルにアンカーされているため、**アンカー先セルの `<td>` 内**にプレースホルダを差し込み、位置を忠実に再現する。
- 対応形式は **PNG / JPEG のみ**。EMF / WMF 等その他形式は**スキップし警告ログ**を出す。

## 4. スコープ外（初版）

以下は初版では対象外とし、将来課題とする。

- グラフ（chart）
- 図形・テキストボックス・矢印などの描画オブジェクト
  - ※ 画面設計書ではテキストボックス・矢印が意味を持つことがあるため、優先度の高い将来課題。
- セルのコメント／メモ
- 見た目書式（色・太字・罫線・フォント）

## 5. 出力構成

```
output/
  {xlsxファイル名}/
    {連番}_{シート名}.md
    images/
      image_1.png
      image_2.png
```

- シート名は `/` や空白等をサニタイズする。
- シート順を保つため連番プレフィックスを付与する。

## 6. インターフェース

- ライブラリ関数を主とし、薄い CLI を付ける。

```python
def convert(xlsx_path: str, out_dir: str, options: Options) -> list[Path]:
    """xlsx を変換し、生成した .md ファイルのパス一覧を返す。"""
```

- オプション例：隠し行／列の除外可否、画像出力先、数式フォールバック挙動 等。

## 7. 依存・技術スタック

- **openpyxl** を主軸（`merged_cells`、書式、画像抽出、`data_only`）。
- 画像抽出は openpyxl の `ws._images` を利用。
- PNG / JPEG 以外の画像はスキップ＋警告。

## 8. 主要な設計判断の記録

| 論点 | 決定 | 理由 |
|---|---|---|
| 用途 | LLM 入力・忠実再現 | レイアウトの位置情報を価値とする |
| 表現 | 1 シート = 1 HTML テーブル | 最も忠実・頑健・実装が素直。LLM は HTML 表を解釈可能 |
| テンプレート | 汎用ヒューリスティック | 様式不定のため意味推定を避ける |
| トリミング | 外周空白のみ | 内部レイアウトを保持 |
| 書式 | 文字値・結合のみ | LLM 入力としてノイズを排除 |
| 画像 | 抽出＋プレースホルダ（アンカー併記） | アンカー先 `<td>` に差し込み位置を忠実化 |

## 9. 想定される将来課題

- 描画オブジェクト（テキストボックス・矢印）の取り込み（画面設計書向け）。
- 巨大スカスカシートのトークン肥大化対策（空白でのブロック分割オプション）。
- グラフの画像化／要約。
- テンプレート判別による意味構造化（フォーム→定義リスト等）の任意モード。
